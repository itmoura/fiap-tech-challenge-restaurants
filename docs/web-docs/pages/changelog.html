🤝 Diretrizes de Contribuição

Para contribuir com o projeto, siga estas diretrizes para manter a qualidade e consistência do código.

📋 Processo de Desenvolvimento

  1. Fork do Repositório: Crie um fork do projeto
  2. Branch de Feature: Crie uma branch para sua feature
  3. Desenvolvimento: Implemente as mudanças seguindo os padrões
  4. Testes: Execute todos os testes e adicione novos se necessário
  5. Documentação: Atualize a documentação conforme necessário
  6. Pull Request: Crie um PR com descrição detalhada

📝 Padrões de Commit

Convenção de Commits 
feat: adiciona nova funcionalidade de busca
fix: corrige problema de validação de dados
docs: atualiza documentação da API
style: formatação de código
refactor: refatoração sem mudança de funcionalidade
test: adiciona ou corrige testes
chore: mudanças em build ou ferramentas
