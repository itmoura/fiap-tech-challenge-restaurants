📚 API Reference - Visão Geral

Documentação completa da REST API do Restaurant Management System

🌐

Informações da API

Base URL http://localhost:8080/api
Versão v1.0.0
Formato JSON
Charset UTF-8
Content-Type application/json
⚡

Quick Start

Teste rapidamente a API com estes exemplos:

🏪 Listar Restaurantes

curl -X GET http://localhost:8080/api/restaurants \
  -H "Content-Type: application/json"

👨‍🍳 Listar Tipos de Cozinha

curl -X GET http://localhost:8080/api/kitchen-types \
  -H "Content-Type: application/json"

🍽️ Criar Novo Restaurante

curl -X POST http://localhost:8080/api/restaurants \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Restaurante Exemplo",
    "description": "Descrição do restaurante",
    "kitchenType": "ITALIAN",
    "address": {
      "street": "Rua Exemplo, 123",
      "city": "São Paulo",
      "state": "SP",
      "zipCode": "01234-567"
    }
  }'

Recursos da API

👨‍🍳

Kitchen Types

Gerenciamento de tipos de cozinha (Italiana, Brasileira, etc.)

🏪

Restaurants

CRUD completo de restaurantes com informações básicas

📂

Menu Categories

Categorias do menu (Entradas, Pratos Principais, etc.)

🍕

Menu Items

Itens específicos do menu com preços e descrições

🔗

Todos os Endpoints

👨‍🍳 Kitchen Types

GET /api/kitchen-types Listar todos os tipos de cozinha
GET /api/kitchen-types/{id} Buscar tipo de cozinha por ID
POST /api/kitchen-types Criar novo tipo de cozinha
PUT /api/kitchen-types/{id} Atualizar tipo de cozinha
DELETE /api/kitchen-types/{id} Remover tipo de cozinha

🏪 Restaurants

GET /api/restaurants Listar todos os restaurantes
GET /api/restaurants/{id} Buscar restaurante por ID
GET /api/restaurants/search Buscar restaurantes por critérios
POST /api/restaurants Criar novo restaurante
PUT /api/restaurants/{id} Atualizar restaurante
DELETE /api/restaurants/{id} Remover restaurante

📂 Menu Categories

GET /api/restaurants/{id}/menu/categories Listar categorias do menu
GET /api/restaurants/{id}/menu/categories/{categoryId} Buscar categoria específica
POST /api/restaurants/{id}/menu/categories Adicionar categoria ao menu
PUT /api/restaurants/{id}/menu/categories/{categoryId} Atualizar categoria
DELETE /api/restaurants/{id}/menu/categories/{categoryId} Remover categoria

🍕 Menu Items

GET /api/restaurants/{id}/menu/categories/{categoryId}/items Listar itens da categoria
GET /api/restaurants/{id}/menu/categories/{categoryId}/items/{itemId} Buscar item específico
POST /api/restaurants/{id}/menu/categories/{categoryId}/items Adicionar item à categoria
PUT /api/restaurants/{id}/menu/categories/{categoryId}/items/{itemId} Atualizar item
DELETE /api/restaurants/{id}/menu/categories/{categoryId}/items/{itemId} Remover item
📄

Formato de Resposta

Todas as respostas da API seguem um formato padrão:

✅ Resposta de Sucesso

{
  "success": true,
  "data": {
    // Dados solicitados
  },
  "message": "Operação realizada com sucesso",
  "timestamp": "2024-08-05T19:30:00Z"
}

❌ Resposta de Erro

{
  "success": false,
  "error": {
    "code": "RESOURCE_NOT_FOUND",
    "message": "Restaurante não encontrado",
    "details": "Nenhum restaurante encontrado com o ID fornecido"
  },
  "timestamp": "2024-08-05T19:30:00Z"
}

📋 Resposta de Lista Paginada

{
  "success": true,
  "data": {
    "content": [
      // Array de itens
    ],
    "pagination": {
      "page": 0,
      "size": 20,
      "totalElements": 150,
      "totalPages": 8,
      "first": true,
      "last": false
    }
  },
  "timestamp": "2024-08-05T19:30:00Z"
}
📊

Códigos de Status HTTP

Código Status Descrição
200 OK Requisição bem-sucedida
201 Created Recurso criado com sucesso
204 No Content Operação bem-sucedida sem conteúdo
400 Bad Request Dados inválidos na requisição
404 Not Found Recurso não encontrado
409 Conflict Conflito com estado atual
500 Internal Server Error Erro interno do servidor
